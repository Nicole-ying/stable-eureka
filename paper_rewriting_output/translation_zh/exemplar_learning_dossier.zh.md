# 范例学习简报（中文）

## 1. 范例清单

| 论文 | 会议 | 年份 | 为何选择 |
|---|---|---|---|
| EUREKA: Human-Level Reward Design via Coding Large Language Models | NeurIPS | 2023 | 直接相关——LLM 作为奖励设计器，迭代式上下文优化。展示了该领域公认的"有效做法"。 |
| Voyager: An Open-Ended Embodied Agent with Large Language Models | NeurIPS | 2023 | 间接相关——LLM 智能体 + 记忆 + 技能库 + 课程学习。展示了 LLM 智能体的结构化记忆模式。 |
| Reflexion: Language Agents with Verbal Reinforcement Learning | NeurIPS | 2023 | 间接相关——语言反思 + 情景记忆以改进决策。展示了 LLM 智能体的反馈循环模式。 |

## 2. 结构模式

**模式 1：范式对比开篇。** 三篇论文都在引言前两段内命名当前范式并指出其局限。EUREKA 的直接对比最为显著（"设计奖励函数是一个劳动密集的试错过程"）。EG-RSA 采用此模式但将对比从"手工设计 vs. LLM 设计"转变为"代码生成 vs. schema 搜索"。

**模式 2：组件逐一阐述。** 每篇方法部分将系统分解为 3-4 个关键组件，每个组件回答"这解决什么问题？"问题。Voyager 的组件阐述最为清晰（自动课程、技能库、迭代提示）。EG-RSA 遵循这一结构：Schema 与归因、记忆、编辑、审计。

**模式 3：失败即洞察的实验弧线。** Reflexion 将失败模式呈现为发现而非 bug。实验不宣称通用优越性——它们展示系统在特定条件下以特定方式工作。EG-RSA 采用此方法：有效编辑 → 退化 → 死锁 → 突破，而非条形图竞赛。

## 3. 修辞模式

**首页推销。** 所有三篇论文在首页完成从问题 → 空白 → 贡献的论证。引言中没有铺垫段落。EG-RSA 必须效仿：首段点明瓶颈，第二段确立 LLM 机会，第三段指出范式局限，第四段提出 EG-RSA，第五段列出贡献。

## 4. 语言模式

- 主动语态贯穿始终（"我们使用"，"系统维护"）
- 无填充短语（无 "值得注意的是"，"有趣的是"）
- 首次引入时使用粗体定义术语
- 贡献编号为 3-4 项，非 5-8 项
- 摘要遵循问题→方法→发现→意义的弧线
