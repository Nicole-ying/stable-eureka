# 从奖励生成到奖励搜索：基于经验引导的语义归因与风险感知编辑的 Schema 自适应

## 摘要

大语言模型辅助的奖励设计近年来成为强化学习的有力工具，但现有方法将奖励设计视为带迭代改进的代码生成——LLM 输出自由形式的奖励代码，标量反馈驱动下一轮迭代。我们认为，奖励设计应被重构为**经验引导的奖励 schema 搜索**：维护跨迭代的版本化、组件化奖励 schema；将策略行为归因至语义奖励角色；将过往编辑结果存储为结构化教训；每次 LLM 提出的编辑必须经过风险审计，并支持自动回滚。我们提出 EG-RSA 并在 LunarLander-v3 上通过机制验证案例研究进行评估。一次有效编辑将主导奖励角色从密集引导转变为终端成功，消除了检测到的失效模式。随后的审计引发死锁揭示了一个根本性张力：阻止有害编辑的安全机制同样会抑制逃离低成功率状态所需的探索。放宽审计策略解除了死锁，为安全奖励搜索中的风险预算机制提供了动力。

## 1. 引言

在强化学习（RL）中，奖励函数定义了优化目标并直接塑造学习行为 \citep{sutton2018reinforcement}。然而，设计有效的奖励函数仍然是一个核心瓶颈：奖励必须提供有用的学习梯度，避免非预期的捷径，并与真实任务目标保持一致，而不仅仅与其相关 \citep{amodei2016concrete,pan2022effects}。

近期研究表明，大语言模型（LLM）能够作为有效的奖励设计器。EUREKA 证明 LLM 能够生成可执行的奖励代码，并通过迭代式上下文优化加以改进 \citep{ma2023eureka}。Text2Reward 从语言描述生成稠密塑形奖励 \citep{xie2023text2reward}。Auto MC-Reward 组合了奖励设计器、评判器和轨迹分析器，从采集的轨迹中精炼奖励 \citep{li2023automcreward}。CARD 提出了带轨迹偏好评估的动态反馈奖励设计 \citep{sun2024card}。这些方法确立了 LLM 作为强大奖励设计工具的地位 \citep{cao2024survey}。

然而，这些方法共享一个共同的范式：它们将奖励设计视为带迭代改进的代码生成。LLM 输出自由形式的奖励代码；代码被训练和评估；下一轮迭代接收标量或文本反馈。这一范式存在结构性局限。跨迭代没有持久的结构化奖励表示——每一轮都从零开始生成代码。没有对奖励\emph{为什么}失败的系统性诊断：标量反馈仅表示性能发生了变化，而无法揭示哪个奖励组件驱动了这种变化。没有过往编辑结果的记忆——每一轮迭代都从空白开始，无法从系统自身有效和退化的编辑历史中学习。没有集成的安全审计：生成的代码在没有执行前风险评估或回滚机制的情况下直接运行。

我们认为，LLM 辅助的奖励设计应被重构为\textbf{经验引导的奖励 schema 搜索}。系统不应从零开始生成完整奖励函数，而应：维护具有语义角色标签的版本化、组件化 schema；通过逐组件归因诊断哪些组件驱动策略行为；从结构化记忆中检索相关的过往编辑结果；将 LLM 约束在可审计的编辑算子内；通过风险审计控制每一次编辑，并支持自动回滚。

我们提出 EG-RSA（Experience-Guided Reward Search Agent，经验引导的奖励搜索智能体），实现了这一重构。从初始奖励 schema 开始，每轮迭代包括：编译 schema，以固定预算训练策略，采集带有逐组件日志的步级轨迹数据，执行语义角色归因，检索相关的成果教训，提示 LLM 生成受约束的编辑方案，审计该方案，并将结果存储为结构化教训。

我们的贡献如下：
1. **经验引导的奖励 schema 搜索。** 我们将 LLM 辅助的奖励设计重构为在结构化、版本化 schema 上的搜索，并带有跨迭代的结果记忆。
2. **通过语义角色归因驱动诊断式编辑。** 带有语义角色标签的逐组件归因将编辑决策建立在诊断证据之上，而非仅依赖标量反馈。
3. **带集成风险审计的算子约束编辑。** LLM 的修改被限制在可审计的算子范围内，并通过行为风险审计进行把关，具有明确的风险分级和回滚机制。

我们在 LunarLander-v3 上通过机制验证案例研究对 EG-RSA 进行评估。我们的核心发现是审计引发的死锁（严格阻断中风险编辑抑制了发现成功行为所需的探索）及其在放宽审计策略下的解除，从而实现了持续改进（任务得分从 0.511 提升至 2.953）。这一发现揭示了安全奖励搜索中的根本性张力，并为平衡安全与探索的风险预算机制提供了动力。

## 2. 相关工作

**RL 中的奖励设计。** 奖励函数是任务意图与策略学习之间的核心接口 \citep{sutton2018reinforcement}。经典奖励塑形研究额外信号如何影响策略学习，其中基于势能的塑形提供了策略不变性的条件 \citep{ng1999policy}。逆强化学习从专家示范中推断奖励 \citep{ng2000irl,abbeel2004apprenticeship}。EG-RSA 在机制和假设上均不同：它既不假设专家示范，也不需要手工设计的塑形势能；而是从训练反馈、归因分析和累积的编辑经验中自适应奖励 schema。

**LLM 辅助的奖励生成。** EUREKA 通过奖励代码的迭代式上下文优化确立了 LLM 作为奖励设计器的地位 \citep{ma2023eureka}。Text2Reward \citep{xie2023text2reward}、Auto MC-Reward \citep{li2023automcreward} 和 CARD \citep{sun2024card} 进一步证明 LLM 可以从语言描述、轨迹分析和动态反馈中生成并精炼稠密奖励函数。一篇近期综述梳理了 LLM 增强 RL 的整体图景 \citep{cao2024survey}。这些方法共享一种代码生成范式。EG-RSA 在四个方面与之不同：(1) 持久版本化 schema 替代了逐轮代码生成；(2) 语义角色归因使诊断驱动的编辑成为可能；(3) 结构化结果教训使搜索具备历史感知能力；(4) 带集成风险审计的可审计算子替代了自由形式的代码生成。

**LLM 智能体中的记忆与反思。** Reflexion 将语言反思存储在情景记忆中，以在不更新权重的情况下改进智能体决策 \citep{shinn2023reflexion}。Voyager 维护一个持续增长的技能库用于终身具身学习 \citep{wang2023voyager}。Generative Agents 使用记忆流实现长程行为 \citep{park2023generative}。一篇近期综述将记忆记录为智能体的核心能力 \citep{zhang2024memorysurvey}。EG-RSA 借鉴了记忆的直觉，但将其应用于一个新的对象：结构化的奖励编辑结果教训，包含 schema 差异、指标增量、失效模式和回滚决策——与语言反思或技能代码不同，这些教训对未来的编辑具有直接的可操作性。

**奖励欺骗与安全性。** 奖励欺骗是一个根本性问题：智能体可能利用代理奖励，而未能完成真实任务 \citep{amodei2016concrete,pan2022effects,skalse2022reward}。近期的缓解方法包括奖励模型集成 \citep{eisenstein2023ensembles}、信息论奖励建模 \citep{lai2024infom}，以及针对重尾误差下 Goodhart 定律的理论分析 \citep{langosco2024goodhart}。LLM 甚至可能从谄媚行为泛化到奖励篡改 \citep{dennison2024sycophancy}。这些工作从奖励设计循环外部诊断和缓解欺骗问题。EG-RSA 将安全性直接集成：行为风险审计在每次训练前对编辑进行把关，具有明确的风险分级和回滚机制。然而，如我们的实验所示，严格审计可能造成死锁，阻断必要的探索——这是事后安全方法不存在的张力。

**定位。** EG-RSA 占据了一个独特位置：它将结构化 schema 表示、语义归因、结果记忆和集成审计组合到单个搜索循环中。这将 LLM 的角色从一次性奖励代码生成器转变为经验引导的奖励搜索算子。

## 3. 方法

### 3.1 搜索循环概览

图 1 展示了一次 EG-RSA 迭代。从一个版本化的奖励 schema 开始，系统：(1) 通过可信编译器将 schema 编译为可执行奖励函数；(2) 使用生成的奖励以固定的充足预算（LunarLander 为 100 万步）训练策略；(3) 采集带有逐组件奖励日志的步级轨迹；(4) 执行语义角色归因，识别主导奖励角色并检测失效模式；(5) 从结构化记忆中检索相关的过往结果教训；(6) 提示 LLM 以 JSON 格式提出受约束的编辑方案；(7) 验证、审计，并决定接受、修复或阻断该编辑；(8) 将结果存储为结构化教训。

策略仅使用生成的奖励进行训练。环境的真实奖励仅用于事后评估——绝不用于奖励选择或编辑。这一分离确保搜索过程不会泄漏真实信息。

EG-RSA 的循环在结构上不同于 Stable-Eureka 及类似方法的基于种群的奖励生成循环。种群方法每轮生成多个完整奖励代码样本，并行训练每个样本，通过真实适应度分数选择最佳样本，并对优胜者进行反思。EG-RSA 将其替换为顺序的 schema 编辑循环：一个 schema 在跨迭代中维护和编辑；LLM 提出结构化编辑而非完全重写；训练反馈通过归因进行诊断而非简化为标量；审计在每次训练前对编辑进行把关。

### 3.2 奖励 Schema 与语义角色归因

EG-RSA 将奖励表示为组件化的 schema，而非无约束的代码。每个组件具有类型、权重、输入变量、参数和启用标志。每个组件被赋予一个语义角色：`dense_guidance`（进度奖励）、`stability_quality`（平滑度和接触质量）、`terminal_success`（任务完成）、`safety_constraint`（违规惩罚）和 `control_cost`（动作惩罚）。一次性条件触发奖励的事件规则也是 schema 元素。

这一表示方式实现了三个能力。第一，版本化：schema 可以被差分、存储和回滚。第二，逐组件归因：由于每个组件在每一步都被单独记录，系统可以测量哪个角色主导了策略行为。第三，算子约束编辑：LLM 使用类型化算子对命名组件提出编辑，而非自由形式的代码。

标量分数本身无法解释奖励为何失败。一个策略可能在稠密进度奖励上得分很高，但从未实现终端成功（*塑形-目标错配*），或通过快速切换接触奖励而不实现稳定着陆（*重复事件利用*）。EG-RSA 从步级轨迹数据中计算逐组件奖励归因：每步组件奖励按回合聚合，主导组件由其占总奖励的比例确定，诊断信号（成功率、接触频率、进度轨迹、奖励重复）组合成失效模式报告。

归因报告——主导角色、失效模式、风险标志——连同当前 schema 和检索到的教训一起提供给 LLM。LLM 将此诊断作为提出编辑的证据，但归因并不机械地决定编辑内容。

### 3.3 结果教训记忆

EG-RSA 存储结构化的结果教训，而非语言反思 \citep{shinn2023reflexion} 或技能代码 \citep{wang2023voyager}。每条教训包含 schema 差异（编辑前后的 schema）、指标增量（任务分数、语义分数、欺骗分数）、编辑前后的失效模式、欺骗风险变化以及回滚决策。教训被分类为 `effective_edit_lesson`（分数提升）或 `regression_lesson`（分数退化）。两种类型都被存储，并根据当前失效模式或 schema 上下文的相似度进行检索。有效教训为哪些编辑策略曾经有效提供了证据；退化教训对先前有害的编辑发出警告。这将前馈式的迭代精炼转变为具有历史感知能力的搜索。

### 3.4 算子约束编辑

LLM 输出一个受限于五种算子的 JSON 编辑方案：`increase_weight`、`decrease_weight`、`add_component`、`add_event_rule` 和 `disable_component`。方案在可信编译器生成可执行奖励代码之前，经过 schema 一致性、算子合法性和规模边界的验证。这一约束使编辑可审计（每次修改都通过已知算子进行）且可逆（回滚是单次 schema 恢复）。

### 3.5 风险感知审计与回滚

每次提出的编辑在训练前都要通过行为风险审计。审计评估三个维度：*欺骗风险*（编辑是否会鼓励先前检测到的失效模式？）、*规模风险*（权重变化是否有破坏训练稳定性的风险？）和*结构风险*（编辑是否移除了安全组件？）。编辑被分类为高、中、低风险。高风险编辑被阻断。低风险编辑通过。中风险编辑根据审计策略处理：在*严格*策略下，当成功证据不足时被阻断；在*放宽*策略下，被接受但带有警告。

训练后，结果验收器检查是否存在退化。如果任务分数大幅下降或已解决的失效模式重新出现，系统回滚到先前的 schema 并记录一条退化教训。否则，新 schema 成为当前版本，并存储一条有效编辑教训。我们实验中的审计规则是为 LunarLander 手工设计的；扩展到新环境需要环境特定的规则或学习到的审计模型。

## 4. 实验

### 4.1 实验设置

我们在 LunarLander-v3 上评估 EG-RSA，这是 Gymnasium 套件中的一个连续控制任务 \citep{towers2024gymnasium}。智能体需要将登月舱平稳降落在着陆平台上，要求低速度、直立姿态和稳定的地面接触（8 维观测，2 维连续动作）。所有实验使用 PPO \citep{schulman2017ppo}，每轮迭代固定 100 万步训练预算，运行 10 轮迭代，使用 DeepSeek 作为 LLM 编辑智能体。所有 EG-RSA 组件均启用：记忆、归因、欺骗检测、算子约束。真实奖励仅用于事后评估。默认审计策略为严格。

**有效编辑（Iter 0→1）。** 在第 0 轮，初始 schema 产生的策略任务得分为 0.478。归因识别出 `r_approach_region`（稠密引导）为主导角色（比率 0.42）。检测到两种失效模式：塑形-目标错配（有进度但没有终端成功）和重复事件利用（快速腿接触切换，平均每回合 370 次事件）。LLM 在了解此诊断且无先前教训（首轮迭代）的情况下，提出：增加 `r_landing_quality` 权重，添加稳定着陆事件规则，降低 `r_approach_region`。该编辑被分类为中风险，但作为首轮例外被允许。训练后，任务得分达到 0.867，`r_landing_quality` 成为主导角色，两种失效模式均消失。系统记录了一条有效编辑教训。

**退化与审计死锁（Iter 1→8）。** 在第 2 轮，任务得分降至 0.412，失效模式回归，`r_approach_region` 再次占主导。系统记录了一条退化教训（指标增量 −0.455）。此次退化后，LLM 反复提出强化终端成功奖励、弱化稠密引导的编辑——与在第 1 轮取得成功的策略相同。然而，这些编辑现在被分类为中风险（修改终端奖励结构），且成功证据不足（自第 1 轮以来没有终端成功）。在严格审计策略下，成功证据不足的中风险编辑被阻断。修复循环无法解决此问题：修复可以调整权重，但无法消除风险分类。结果是第 2-8 轮的死锁：得分维持在 0.367-0.665 之间，失效模式持续存在，系统无法逃脱。

这一死锁并非实现缺陷——它揭示了一个根本性的设计张力。防止有害编辑的同一审计机制，同样可能阻断建立成功证据所需的探索。系统需要修改终端奖励来发现成功着陆，但在成功证据缺失的情况下，严格审计恰好阻断了这些修改。

| 3 | 2.953 | 4.453 | 0.00 | (无) | r_landing_quality |

*表 2：放宽审计下的 EG-RSA。*

从相同的初始 schema（0.478）开始，放宽审计运行走上了不同的轨迹。在第一次温和编辑（0.511）后，第 2 轮实现了突破：任务得分 2.490，无失效模式，`r_landing_quality` 占主导。第 3 轮继续提升（2.953）。这些结果证实死锁是由审计引发的：相同的搜索机制——归因、记忆、约束编辑——在审计门槛降低后产生了持续改进。

这些结果并不意味着应该移除审计。相反，它们为风险预算机制提供了动力：高风险编辑仍然被阻断，中风险编辑在监控下被允许，回滚作为安全网。设计这样的机制是未来工作。

## 5. 讨论

这些实验作为机制验证，而非性能基准。三个发现浮出水面。第一，归因引导的、带有结构化记忆的编辑可以产生有效的奖励修改，如第 0→1 轮案例研究所示。第二，退化在奖励搜索中是真实存在的，系统通过结构化结果教训对其进行了跟踪。第三，审计死锁揭示了一个根本性张力：将安全约束集成到奖励搜索中会在风险防范和探索之间产生权衡，严格阻断可能抑制建立成功证据所需的编辑。

我们的研究存在若干局限。我们仅在单一环境（LunarLander-v3）上进行了评估。任务指标是模板实例化的，而非自动发现的。审计规则为该环境手工设计。实验运行 10 轮迭代；更长时程的动态尚未探索。我们没有与 EUREKA 或其他方法进行基准比较——这种比较需要超出本机制验证范围的多环境评估。

审计死锁的发现具有超出 EG-RSA 的意义。任何引入安全约束的基于 LLM 的奖励设计系统——无论是基于规则、学习还是集成——都面临同样的权衡。在弱证据下严格阻断中风险修改可能抑制了首先建立这些证据所需的编辑。

未来工作包括多环境评估、适配环境特定失效模式的学习型审计规则、风险预算机制的实现，以及集成过程级奖励模型 \citep{zheng2025prmsurvey} 以细化归因粒度。

## 6. 结论

我们将 LLM 辅助的奖励设计重构为经验引导的 schema 搜索，引入了语义角色归因、结构化结果记忆和带集成的风险审计与回滚。在 LunarLander-v3 上的机制验证实验展示了有效编辑、退化跟踪，以及一个揭示安全-探索权衡的审计引发死锁。通过放宽审计解除这一死锁，为安全奖励搜索中的风险预算机制提供了未来工作方向。
