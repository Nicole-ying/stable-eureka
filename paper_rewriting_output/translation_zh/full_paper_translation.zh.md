# 从奖励生成到奖励搜索：基于语义归因与风险感知编辑的经验引导 Schema 自适应

## 摘要

大语言模型辅助的奖励设计近年来成为减少强化学习中手动奖励工程的一种有前景的方法。现有方法表明，大语言模型能够根据任务描述和训练反馈生成并精炼可执行的奖励代码。然而，可靠的奖励设计往往需要的不仅仅是奖励生成：系统必须诊断奖励为何失败，记住哪些先前的编辑有效或有害，将策略行为与奖励组件关联起来，并在存在奖励欺骗风险的情况下验证拟议的编辑。我们提出 EG-RSA，一个经验引导的奖励 Schema 自适应（Experience-Guided Reward Schema Adaptation）框架，将 LLM 辅助的奖励设计形式化为历史感知的奖励 schema 搜索。EG-RSA 将奖励表示为具有语义角色的结构化 schema，将策略行为归因至奖励组件，将有效和退化的编辑存储为结果教训，并通过可审计的编辑算子约束 LLM 的修改，辅以风险感知验证和回滚机制。我们在 LunarLander-v3 上以机制验证基准的方式研究 EG-RSA，分析了有效编辑、退化教训以及行为风险审计引入的安全-探索权衡。

## 1. 引言

强化学习（RL）在序贯决策和控制任务中取得了显著成果，但其成功仍然高度依赖定义学习目标的奖励函数 \citep{sutton2018reinforcement}。设计有效的奖励是困难的：奖励必须提供有用的学习信号，鼓励任务完成，并与预期目标保持一致。设计不当的奖励可能导致非预期行为，包括奖励欺骗、过度利用稠密塑形信号，或代理奖励高而真实任务表现差 \citep{amodei2016concrete,pan2022effects}。

近期研究探索了将大语言模型（LLM）作为自动化奖励设计器。Eureka 证明 LLM 能够生成可执行的奖励代码，并通过迭代式上下文优化加以改进 \citep{ma2023eureka}。Text2Reward 从自然语言任务描述生成稠密奖励函数 \citep{xie2023text2reward}。Auto MC-Reward 利用轨迹分析和奖励评判器在稀疏奖励的 Minecraft 任务中精炼奖励 \citep{li2023automcreward}。CARD 进一步研究了面向 LLM 驱动奖励设计的动态反馈和轨迹偏好评估 \citep{sun2024card}。这些方法表明 LLM 能够减轻手动奖励工程的负担。

然而，可靠的奖励设计需要的不仅仅是生成或精炼奖励代码。当奖励失败时，关键问题往往不仅是如何重写它，而是它为何失败：哪个奖励组件主导了策略行为，哪个先前的编辑改善或恶化了策略，以及新编辑是否类似于先前有害的搜索方向。现有基于 LLM 的奖励设计方法提供了有价值的反馈循环，但它们没有明确将奖励设计建模为基于累积编辑经验的结构化搜索过程。

我们认为，LLM 辅助的奖励设计应被形式化为经验引导的奖励 schema 搜索。系统不应反复生成自由形式的奖励代码，而应在迭代过程中维护结构化的奖励 schema，通过语义奖励角色诊断策略失败，将有效和退化的编辑存储为可复用的结果教训，并在应用前验证每个 LLM 提议的编辑。

我们提出 EG-RSA，一个面向 LLM 辅助强化学习的经验引导奖励 Schema 自适应框架。EG-RSA 将奖励表示为具有语义角色的组件化 schema，如稠密引导、稳定性质量、终端成功、安全约束和控制成本。在每次训练迭代后，EG-RSA 采集轨迹反馈，计算语义结果，将策略行为归因至奖励角色，检索相关的结果教训，并要求 LLM 提出受算子约束的奖励编辑。该编辑随后经过规模验证、行为风险审计、修复和回滚机制的检查。

我们的贡献有三点：

1. 我们将 LLM 辅助的奖励设计形式化为经验引导的奖励 schema 搜索，其中奖励以结构化、版本化的 schema 方式维护，而非重新生成为无约束的代码。
2. 我们引入语义角色归因和结果教训记忆，以连接策略失败与奖励组件，并在迭代间复用有效或退化的奖励编辑经验。
3. 我们设计了具有验证和回滚机制的风险感知算子约束编辑循环，并分析了当审计规则在低成功率状态下过于保守时产生的安全-探索权衡。

## 2. 相关工作

### 2.1 奖励设计与奖励塑形

奖励设计是强化学习中一个长期存在的挑战。由于奖励函数定义了优化目标，学习到的策略对任务意图如何转化为标量反馈高度敏感 \citep{sutton2018reinforcement}。奖励塑形旨在通过添加辅助信号来提高学习效率，基于势能的塑形提供了奖励变换保持最优策略的条件 \citep{ng1999policy}。然而，这些结果也凸显了奖励工程的困难：塑形信号必须精心设计，以加速学习而不改变预期行为。

另一条研究线索是从示范中学习奖励。逆强化学习从专家行为中推断奖励函数 \citep{ng2000irl}，学徒学习利用这些推断出的目标来模仿专家策略 \citep{abbeel2004apprenticeship}。这些方法减少了手动奖励定义的工作量，但通常需要示范或关于专家行为的假设。EG-RSA 针对不同的设定：它不假设专家示范，而是通过策略训练反馈、语义诊断和累积的奖励编辑经验来自适应初始奖励 schema。

### 2.2 LLM 辅助的奖励设计

大语言模型最近被用于自动化奖励设计。Eureka 证明 LLM 能够生成可执行的奖励代码，并通过迭代式上下文优化加以改进 \citep{ma2023eureka}。Text2Reward 从自然语言任务描述生成塑形稠密奖励 \citep{xie2023text2reward}。Auto MC-Reward 组合了奖励设计器、奖励评判器和轨迹分析器，从轨迹反馈中精炼稠密奖励 \citep{li2023automcreward}。CARD 进一步引入了动态反馈和轨迹偏好评估，以提高 LLM 驱动奖励精炼的效率 \citep{sun2024card}。一篇近期综述总结了 LLM 增强的 RL，并将奖励设计确定为 LLM 的关键角色之一 \citep{cao2024survey}。

这些方法确立了 LLM 作为有效奖励设计器的地位。EG-RSA 沿袭这一方向，但聚焦于一个不同的问题：奖励搜索经验如何能够跨迭代存储和复用？EG-RSA 不将每次奖励更新视为孤立的代码生成步骤，而是维护版本化的奖励 schema，将策略行为归因至语义奖励角色，将有效和退化的编辑记录为结果教训，并将 LLM 的修改限制在可审计的算子范围内。这将奖励设计从奖励代码生成转变为结构化的、历史感知的奖励 schema 搜索。

### 2.3 奖励搜索中的记忆、反思与可靠性

EG-RSA 与基于记忆和反思的语言智能体相关。Reflexion 通过语言反馈和情景记忆改进语言智能体，而无需更新模型参数 \citep{shinn2023reflexion}。Voyager 为终身具身学习维护技能库 \citep{wang2023voyager}，Generative Agents 则利用记忆流和反思支持长程行为 \citep{park2023generative}。这些工作表明 LLM 智能体可以从存储和复用过往经验中获益。EG-RSA 将这一思想应用于奖励设计，但存储的是结构化的奖励编辑结果教训，而非自然语言反思或可执行的行动技能。

可靠性对奖励设计也至关重要。奖励欺骗和奖励错误定义发生在智能体优化代理奖励而未能完成预期目标的情况下 \citep{amodei2016concrete,pan2022effects,skalse2022reward}。这一问题对于 LLM 生成的奖励尤为相关，其中稠密塑形项可能易于优化但与稀疏的终端成功不一致。EG-RSA 通过失效模式诊断、语义角色归因、行为风险审计和回滚来解决此问题。同时，我们的分析表明，安全机制必须与探索取得平衡：如果风险审计过于保守，可能阻断逃离低成功率状态所需的编辑。

## 3. 方法

### 3.1 问题形式化与框架概述

我们考虑一个 RL 环境，其中策略使用从结构化奖励 schema 编译而来的奖励函数进行训练。令 $S_k$ 表示第 $k$ 轮迭代的奖励 schema，令 $R_{S_k}$ 为从该 schema 编译得到的可执行奖励函数。EG-RSA 的主要贡献并非直接优化策略参数；而是搜索一系列奖励 schema $\{S_k\}_{k=0}^{K}$，使得在编译奖励下训练的策略越来越符合预期的任务目标。

给定初始奖励 schema $S_0$、任务描述、诊断规范和记忆存储，EG-RSA 执行迭代式奖励 schema 搜索。在每轮迭代中，它将当前 schema 编译为奖励代码，使用编译后的奖励训练策略，采集语义反馈和轨迹诊断信息，将策略行为归因至奖励角色，检索相关的历史教训，要求 LLM 编辑智能体提出受约束的编辑方案，并通过风险感知审计和回滚来验证该编辑。

图 1 展示了该框架。与一次性奖励生成的关键区别在于，EG-RSA 维护了版本化的奖励 schema 和跨迭代的先前编辑结果记忆。因此，LLM 被用作奖励搜索算子，而非自由形式的奖励代码生成器。

### 3.2 奖励 Schema 与编辑算子

EG-RSA 将奖励函数表示为奖励 schema
$$S = (\mathcal{C}, \mathcal{E}, \mathcal{M}),$$
其中 $\mathcal{C}=\{c_i\}_{i=1}^{n}$ 是一组稠密奖励组件，$\mathcal{E}=\{e_j\}_{j=1}^{m}$ 是一组基于事件的奖励规则，$\mathcal{M}$ 包含语义角色等元数据。每个稠密组件具有名称、权重、度量函数和语义角色。典型角色包括 `dense_guidance`、`stability_quality`、`terminal_success`、`safety_constraint` 和 `control_cost`。

从 schema $S$ 编译得到的奖励函数可写为
$$R_S(s_t,a_t,s_{t+1}) = \sum_{i=1}^{n} w_i \phi_i(s_t,a_t,s_{t+1}) + \sum_{j=1}^{m} \beta_j \mathbf{1}[e_j(s_t,a_t,s_{t+1})],$$
其中 $\phi_i$ 是稠密度量函数，$w_i$ 是其权重，$e_j$ 是事件条件，$\beta_j$ 是事件奖励，$\mathbf{1}[\cdot]$ 是指示函数。

EG-RSA 不允许 LLM 自由重写整个奖励函数，而是将奖励修改限制为算子约束的编辑方案
$$\Delta_k = \{o_1,o_2,\ldots,o_p\},$$
其中每个算子属于预定义的集合，包括 `increase_weight`、`decrease_weight`、`add_component`、`add_event_rule` 和 `disable_component`。候选 schema 由以下公式得到
$$S'_k = \mathrm{Apply}(S_k,\Delta_k).$$
这一设计使编辑可检查、可逆且可审计。

### 3.3 语义反馈与角色归因

在当前 schema 下训练策略后，EG-RSA 使用语义结果和轨迹诊断来评估策略行为。这些信号包括任务进度、成功率、终端奖励触发率、稳定着陆证据、安全接触证据、回合长度、接触不稳定性和奖励欺骗指标。

仅凭标量分数无法解释奖励为何失败。策略可能在稠密进度奖励上得分很高但从未触发终端成功，或通过不稳定的重复事件利用接触相关奖励。为诊断此类失败，EG-RSA 从记录的逐组件奖励中计算逐组件奖励归因。框架识别主导奖励组件，将其映射到语义角色，并将此归因与失效模式诊断相结合。

例如，如果稠密接近组件主导了总奖励而终端成功保持为零，则策略可能在利用稠密引导而未完成任务。这样的诊断为 LLM 编辑智能体提供了关于哪些奖励角色应该被削弱、保留或由新事件规则补充的证据。

### 3.4 经验记忆与编辑规划

EG-RSA 维护一个奖励搜索经验的记忆存储。在每次编辑被评估后，框架记录一条结果教训
$$\ell_k = (S_k, \Delta_k, S'_k, O_k, O'_k, F_k, d_k),$$
其中 $O_k$ 和 $O'_k$ 是编辑前后的语义结果，$F_k$ 包含失效模式诊断，$d_k$ 是接受、拒绝或回滚决策。如果编辑改善了任务行为或降低了风险，则存储为有效教训。如果编辑导致退化或增加了奖励欺骗风险，则存储为退化教训。

在第 $k$ 轮迭代，EG-RSA 从记忆中检索相关教训，并将其与当前 schema、语义结果、失效模式和归因报告一起纳入 LLM 编辑提示。这使得 LLM 能够基于累积的奖励搜索历史来条件化其提议的编辑，而非仅依赖当前标量分数。

### 3.5 风险感知验证与回滚

每个提议的编辑在应用之前都需要经过验证。EG-RSA 首先检查算子合法性和 schema 一致性。然后执行规模验证以避免破坏稳定性的权重变化。最后，行为风险审计检查该编辑是否可能放大稠密奖励主导性、在充分证据之前过度强调终端奖励、移除重要的稳定性组件，或与先前的退化教训相似。

编辑按风险级别分类。高风险编辑被阻断或送修。低风险编辑通过。中风险编辑由审计策略控制。严格策略可能在成功证据不足时阻断中风险编辑，而放宽策略可以允许它们带有警告，并依赖后续的结果评估和回滚。这一区分很重要，因为过于保守的审计可能抑制逃离低成功率状态所需的探索。

训练候选 schema 后，EG-RSA 应用结果验收器。如果候选改善了相关的语义指标且未增加欺骗风险，则候选 schema 成为下一个 schema。否则，系统回滚到先前的最佳 schema，并将失败的编辑存储为退化教训。

### 3.6 算法总结

算法 1 总结了 EG-RSA。该框架形成了策略训练、语义诊断、记忆检索、LLM 编辑规划和风险感知验证之间的闭环。

**算法 1：EG-RSA 奖励 Schema 搜索**

**输入：** 初始 schema $S_0$，任务描述 $T$，诊断规范 $D$，记忆存储 $\mathcal{L}$，迭代次数 $K$

对 $k=0$ 到 $K-1$ 执行：
1. 从 schema $S_k$ 编译奖励函数 $R_{S_k}$
2. 在奖励 $R_{S_k}$ 下使用 RL 训练策略 $\pi_k$
3. 采集语义结果 $O_k$ 和轨迹诊断 $F_k$
4. 计算语义奖励角色归因 $A_k$
5. 从记忆中检索相关的结果教训 $\mathcal{L}_k$
6. 使用 $(T,S_k,O_k,F_k,A_k,\mathcal{L}_k)$ 构建编辑提示
7. 查询 LLM 编辑智能体产生编辑方案 $\Delta_k$
8. 使用算子约束和规模审计验证 $\Delta_k$
9. 使用 $(F_k,A_k,\mathcal{L}_k)$ 执行行为风险审计
10. 如果 $\Delta_k$ 通过验证和审计：
    - $S'_k \gets \mathrm{Apply}(S_k,\Delta_k)$
    - 训练并评估候选 schema $S'_k$
    - 如果候选结果被接受：
      - $S_{k+1} \gets S'_k$
      - 在 $\mathcal{L}$ 中存储有效结果教训
    - 否则：
      - $S_{k+1} \gets$ 回滚 schema
      - 在 $\mathcal{L}$ 中存储退化教训
11. 否则：
    - 修复 $\Delta_k$ 或继续使用 $S_k$
    - $S_{k+1} \gets S_k$

**返回** 最佳 schema $S^\star$

## 4. 实验

**有效编辑（Iter 0→1）。** 在第 0 轮，初始 schema 产生的策略任务得分为 0.478。归因识别出 `r_approach_region`（稠密引导）为主导角色（比率 0.42）。检测到两种失效模式：塑形-目标错配（有进度但没有终端成功）和重复事件利用（快速腿接触切换，平均每回合 370 次事件）。LLM 在了解此诊断且无先前教训（首轮迭代）的情况下，提出：增加 `r_landing_quality` 权重，添加稳定着陆事件规则，降低 `r_approach_region`。该编辑被分类为中风险，但作为首轮例外被允许。训练后，任务得分达到 0.867，`r_landing_quality` 成为主导角色，两种失效模式均消失。系统记录了一条有效编辑教训。

**退化与审计死锁（Iter 1→8）。** 在第 2 轮，任务得分降至 0.412，失效模式回归，`r_approach_region` 再次占主导。系统记录了一条退化教训（指标增量 −0.455）。此次退化后，LLM 反复提出强化终端成功奖励、弱化稠密引导的编辑——与在第 1 轮取得成功的策略相同。然而，这些编辑现在被分类为中风险（修改终端奖励结构），且成功证据不足（自第 1 轮以来没有终端成功）。在严格审计策略下，成功证据不足的中风险编辑被阻断。修复循环无法解决此问题：修复可以调整权重，但无法消除风险分类。结果是第 2-8 轮的死锁：得分维持在 0.367-0.665 之间，失效模式持续存在，系统无法逃脱。

这一死锁并非实现缺陷——它揭示了一个根本性的设计张力。防止有害编辑的同一审计机制，同样可能阻断建立成功证据所需的探索。系统需要修改终端奖励来发现成功着陆，但在成功证据缺失的情况下，严格审计恰好阻断了这些修改。

### 4.3 放宽审计策略

为检验死锁是否由审计引发，我们使用放宽策略进行了第二次实验：弱证据下的中风险编辑被接受但带有警告。其他设置完全相同。

| Iter | Task | Semantic | Hack | Failures | Dominant Component |
|------|------|----------|------|----------|-------------------|
| 0 | 0.478 | 0.478 | 0.40 | REE, SGM | r_approach_region |
| 1 | 0.511 | 0.511 | 0.40 | REE, SGM | r_approach_region |
| 2 | 2.490 | 3.907 | 0.00 | (无) | r_landing_quality |
| 3 | 2.953 | 4.453 | 0.00 | (无) | r_landing_quality |

*表 2：放宽审计下的 EG-RSA。*

从相同的初始 schema（0.478）开始，放宽审计运行走上了不同的轨迹。在第一次温和编辑（0.511）后，第 2 轮实现了突破：任务得分 2.490，无失效模式，`r_landing_quality` 占主导。第 3 轮继续提升（2.953）。这些结果证实死锁是由审计引发的：相同的搜索机制——归因、记忆、约束编辑——在审计门槛降低后产生了持续改进。

这些结果并不意味着应该移除审计。相反，它们为风险预算机制提供了动力：高风险编辑仍然被阻断，中风险编辑在监控下被允许，回滚作为安全网。设计这样的机制是未来工作。

## 5. 讨论

这些实验作为机制验证，而非性能基准。三个发现浮出水面。第一，归因引导的、带有结构化记忆的编辑可以产生有效的奖励修改，如第 0→1 轮案例研究所示。第二，退化在奖励搜索中是真实存在的，系统通过结构化结果教训对其进行了跟踪。第三，审计死锁揭示了一个根本性张力：将安全约束集成到奖励搜索中会在风险防范和探索之间产生权衡，严格阻断可能抑制建立成功证据所需的编辑。

我们的研究存在若干局限。我们仅在单一环境（LunarLander-v3）上进行了评估。任务指标是模板实例化的，而非自动发现的。审计规则为该环境手工设计。实验运行 10 轮迭代；更长时程的动态尚未探索。我们没有与 EUREKA 或其他方法进行基准比较——这种比较需要超出本机制验证范围的多环境评估。

审计死锁的发现具有超出 EG-RSA 的意义。任何引入安全约束的基于 LLM 的奖励设计系统——无论是基于规则、学习还是集成——都面临同样的权衡。在弱证据下严格阻断中风险修改可能抑制了首先建立这些证据所需的编辑。

未来工作包括多环境评估、适配环境特定失效模式的学习型审计规则、风险预算机制的实现，以及集成过程级奖励模型 \citep{zheng2025prmsurvey} 以细化归因粒度。

## 6. 结论

我们将 LLM 辅助的奖励设计重构为经验引导的 schema 搜索，引入了语义角色归因、结构化结果记忆和带集成的风险审计与回滚。在 LunarLander-v3 上的机制验证实验展示了有效编辑、退化跟踪，以及一个揭示安全-探索权衡的审计引发死锁。通过放宽审计解除这一死锁，为安全奖励搜索中的风险预算机制提供了未来工作方向。
