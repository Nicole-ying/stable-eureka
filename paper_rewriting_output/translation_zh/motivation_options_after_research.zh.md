# 候选动机（研究后，中文）

## 选项 A：经验引导的奖励 schema 搜索

**一句话动机：** LLM 辅助的奖励设计应被构建为经验引导的奖励 schema 搜索——而非一次性代码生成。

**核心创新：** 将奖励设计从逐轮代码生成重构为通过结构化、版本化 schema 进行的搜索，具有语义角色标签、逐组件行为归因、结构化结果教训记忆，以及带自动回滚的集成风险审计。

**为何不是过于宽泛：** 我们不声称解决奖励设计。我们声称搜索重构产生新的能力（历史感知编辑、诊断驱动修改、审计把关）和新的发现（审计死锁作为安全-探索张力）。

**所需证据：**
- 有效编辑案例研究（E1：Iter 0→1，0.478→0.867）展示归因 → 编辑 → 改进因果链
- 退化教训存储（E1：Iter 1→2，0.867→0.412）展示结构化记忆捕获失败
- 审计死锁 + 放宽突破（E1+E2：死锁 0.367-0.665 持续 7 轮，放宽后突破至 2.953）
- 架构对比（设计文档 S2：EG-RSA 循环 vs. Stable-Eureka 循环，5 个结构差异）
- 代码实现（E6：约束编辑 schema、审计规则、记忆存储）

**最佳论文弧线：** 范式 → 局限 → 搜索重构 → 组件 → 案例研究 → 张力 → 意义
