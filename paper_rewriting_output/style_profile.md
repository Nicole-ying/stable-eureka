# Style Profile

| Style Dimension | Target Venue Expectation | Exemplar Pattern | Applied To This Paper |
|---|---|---|---|
| **Register** | Formal academic English; first-person plural ("we"); active voice; terse over ornate. Conference papers are shorter — every sentence carries argument weight. | EUREKA, Voyager, Reflexion all use consistent "we" + active constructions. No passive hedging for implemented mechanisms. | Use "we" throughout. Active voice for mechanisms ("EG-RSA constrains," "we observe"). Eliminate filler: no "it is worth noting," no "interestingly," no "in order to." |
| **Opening strategy** | Introduction narrows from field problem → paradigm limitation → specific gap → proposed mechanism. First paragraph must name the bottleneck concretely. | All three exemplars open with a concrete problem statement, not "AI is important." EUREKA: "designing reward functions is a labor-intensive trial-and-error process." | Open with: reward design is a central bottleneck; LLM reward generation exists but follows a one-shot/free-code paradigm; what's missing is history-aware, auditable reward search. |
| **Method exposition** | Component-by-component with closing integration paragraph. Each subsection names a mechanism, explains its function, and shows how it addresses a specific failure mode. | EUREKA's method sections each answer "what problem does this solve?" before describing how. Voyager's integration paragraph shows the full loop. | Structure Method as: Schema → Attribution → Memory → Editing → Audit → Integration paragraph showing the full iteration. |
| **Claim calibration** | Mechanisms are stated directly ("EG-RSA stores"), interpretive findings are hedged ("suggests," "is consistent with"). Contributions are mechanisms, not environment counts. | EUREKA lists 3 contributions; Voyager lists 4. None count environments as contributions. | List 3–4 contributions: (1) reward search framing, (2) semantic role attribution + memory, (3) operator-constrained editing with audit. |
| **Experiment framing** | Journal reviewers expect mechanism verification, not benchmark domination for a new framework. Case studies and diagnostic traces are stronger than single aggregate scores. | Voyager analyzes curriculum gaps; Reflexion uses minimal environments to verify the reflection loop. | Frame the 10-iteration LunarLander run as mechanism verification with three case studies: effective edit, regression + rollback, audit deadlock. |
| **Figure usage** | Every figure referenced before appearance. Captions are self-contained summaries, not descriptions. Vector format preferred. | EUREKA's workflow diagram is referenced in the first paragraph of Method and explained before it appears. | Reference the EG-RSA loop diagram early in Method. Captions: "EG-RSA search loop showing one full iteration from schema to outcome lesson." |
| **Related Work** | Each subsection ends with explicit contrast to EG-RSA, not just a summary. The section builds toward a "positioning" paragraph. | EUREKA's Related Work has a dedicated "differences from prior work" paragraph. | Related Work closes each subsection with "In contrast, EG-RSA..." and ends with a Positioning subsection. |
| **Limitations** | Dedicated paragraph or subsection in Discussion. Specific, not vague. Connected to future work. | Voyager lists 3–4 specific constraints. | List: (1) single-environment verification, (2) template-instantiated task metrics, (3) audit rules are hand-designed, (4) 10-iteration scale. |
| **Abstract** | 150–250 words. Problem → gap → method → key findings → implication. No citations. | All three exemplars use this structure. | Follow the 5-sentence arc: (1) reward design bottleneck, (2) current LLM methods follow generation paradigm, (3) EG-RSA reformulates as search, (4) mechanisms, (5) findings from case studies. |
