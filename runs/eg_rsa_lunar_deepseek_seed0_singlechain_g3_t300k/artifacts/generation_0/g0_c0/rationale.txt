This reward function implements the required schema components exactly as specified. Each component is computed from the observable next_obs and action inputs without using any hidden implementation details. The distance, velocity, and angle penalties are simple negative magnitudes that guide the lander toward the pad with gentle speed and upright orientation. The engine usage penalty discourages unnecessary main engine firing (-0.3 per use) to promote fuel efficiency. The terminal reward (+100) is triggered only when the lander achieves a successful landing: both legs grounded, close to pad (<0.1), upright (<0.1 rad), and low vertical speed (<0.1 normalized). The components are linearly combined and the total is bounded well under 1000 for typical episodes.
