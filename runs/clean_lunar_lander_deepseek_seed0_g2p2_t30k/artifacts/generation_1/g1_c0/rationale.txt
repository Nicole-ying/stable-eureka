This reward function addresses the failure modes from the previous generation. The distance penalty is reduced from quadratic (-0.5*d²) to linear (-0.1*d) to avoid overwhelming other signals. The landing bonus is dramatically increased to +150, making successful landing the dominant objective. All terminal logic is consolidated into a single `if done:` block to avoid double-counting. Progress shaping provides positive rewards for moving toward the pad, reducing vertical speed, and becoming upright. The effort penalty uses a looser threshold (dist < 0.3) and is very small. No imports are used. Component magnitudes are balanced so no single component dominates except the landing bonus, which is intentional to prioritize landing.
