## Search Plan for Reward Generation

1. **Initialize with baseline components**: Use the six components defined in the schema. For each component, define candidate functional forms (linear, quadratic, exponential, clipped) and scaling factors.

2. **Generate variations**: For each component, create 3-5 variants by:
   - Adjusting the functional form (e.g., `-distance`, `-distance^2`, `-exp(distance)`)
   - Tuning the coefficient magnitude to balance component scales (target: each component contributes roughly 0.1 to 10 per step)
   - Adding clipping or thresholds (e.g., only penalize velocity above a threshold)

3. **Combine components**: Assemble full reward functions by selecting one variant per component. Use additive combination with optional multiplicative gating (e.g., only apply angle penalty when in air).

4. **Evaluate candidate rewards**: For each candidate reward function, run a short training (e.g., 50k steps) with a simple RL algorithm (e.g., PPO). Track:
   - Episodic return
   - Landing success rate
   - Average fuel consumption
   - Crash rate

5. **Iterate**: Based on evaluation, refine component forms and coefficients. Re-run top candidates with longer training (e.g., 200k steps). Select the best-performing reward function.

6. **Final validation**: Test the chosen reward on multiple random seeds to ensure robustness. Report the final reward function and its performance metrics.
