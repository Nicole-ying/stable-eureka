## Search Plan for Reward Generation

### Objective
Design a reward function for the LunarLander environment that encourages fast, fuel-efficient, and stable landings at the pad (0,0). The reward must be computed solely from `state`, `m_power`, `s_power`, and `terminated`.

### Approach
1. **Component Design**: Use the five components defined in the schema: landing_bonus, velocity_penalty, angle_penalty, fuel_efficiency, and distance_progress. Each component will be a function of the observable variables.
2. **Initial Coefficients**: Start with simple linear or quadratic penalties with hand-tuned weights. For example:
   - landing_bonus: +100 if conditions met, else 0.
   - velocity_penalty: -k1*|vx| - k2*|vy|.
   - angle_penalty: -k3*|angle| - k4*|angular_vel|.
   - fuel_efficiency: -k5*(m_power + s_power).
   - distance_progress: -k6*sqrt(x^2 + y^2).
3. **Iterative Tuning**: Run simulations to observe behavior and adjust weights to balance trade-offs (e.g., prevent hovering vs. crashing).
4. **Normalization**: Scale components to have comparable magnitudes (order of 1-10) to avoid dominance.
5. **Terminal Condition**: The landing_bonus is the terminal-shaping component; no separate terminal penalty is needed.
6. **Testing**: Evaluate on multiple random seeds to ensure robustness to wind and initial conditions.

### Expected Outcome
A reward function that yields stable landings with low fuel usage and minimal episode length.
