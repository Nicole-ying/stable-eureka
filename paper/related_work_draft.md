## 2.1 Reward Design and Reward Shaping in Reinforcement Learning

Reinforcement learning formulates sequential decision making as an interaction between an agent and an environment, where the reward function defines the optimization objective and directly shapes the learned policy behavior [@sutton2018reinforcement]. In practice, however, designing an effective reward function is often difficult. A reward signal must not only encourage task completion, but also provide useful learning gradients, avoid misleading shortcuts, and remain aligned with the true task objective. This makes reward design a central bottleneck in applying reinforcement learning to complex control tasks.

Classical reward shaping studies how additional reward signals can accelerate learning. Potential-based reward shaping provides theoretical conditions under which transformed rewards preserve the optimal policy [@ng1999policy]. This line of work shows that shaping rewards can be useful, but also highlights a key challenge: poorly designed shaping signals may change what the agent learns or create unintended local optima. Another related direction is inverse reinforcement learning, which aims to infer reward functions from expert demonstrations [@ng2000irl; @abbeel2004apprenticeship]. These methods reduce the need for hand-written rewards, but they often assume access to demonstrations or expert behavior.

EG-RSA differs from these traditional approaches in its problem setting and mechanism. It does not assume expert demonstrations, nor does it rely on a fixed hand-designed shaping potential. Instead, EG-RSA starts from an initial reward schema and iteratively adapts it using policy training feedback, semantic failure diagnosis, reward role attribution, and accumulated editing experience. In our experiments, PPO is used as the underlying policy optimizer [@schulman2017ppo], while the main focus of this work is the orthogonal problem of how reward functions can be searched, edited, and validated.

## 2.2 LLM-Assisted Reward Generation

Recent work has explored large language models as reward designers. Eureka demonstrates that LLMs can generate executable reward code and improve it through iterative in-context optimization over reward programs [@ma2023eureka]. It shows that LLM-generated rewards can outperform human-engineered rewards in a wide range of reinforcement learning environments. Text2Reward uses language models to generate dense reward functions from natural language task descriptions and supports refinement with human feedback [@xie2023text2reward]. Auto MC-Reward further applies LLM-based dense reward design to sparse-reward Minecraft tasks, combining a Reward Designer, Reward Critic, and Trajectory Analyzer to refine reward functions from collected trajectories [@li2023automcreward]. CARD proposes a dynamic-feedback framework for LLM-driven reward design, using a Coder and Evaluator together with Trajectory Preference Evaluation to reduce unnecessary RL training during reward refinement [@sun2024card].

These methods establish that LLMs can be effective tools for reward generation and iterative reward improvement. However, many of them primarily treat reward design as a reward code generation or refinement problem. In contrast, EG-RSA explicitly models reward design as an experience-guided reward schema search problem. Rather than allowing the LLM to freely rewrite the entire reward function, EG-RSA represents rewards as semantically annotated schemas and restricts modifications to auditable edit operators. More importantly, EG-RSA records both effective edits and regressive edits as structured outcome lessons, allowing later iterations to retrieve and reuse prior search experience.

This distinction is important because reward design failures are often not isolated one-step errors. A policy may appear to improve under dense shaping rewards while failing the true task objective, or a reward modification may improve short-term progress while increasing reward hacking risk. EG-RSA therefore focuses not only on generating reward code, but also on diagnosing why a reward failed, which reward role dominated the learned behavior, whether a proposed edit resembles prior regression patterns, and whether the edit should be accepted, repaired, or rolled back.

## 2.3 Memory and Reflection in Language Agents

EG-RSA is also related to memory-based and reflection-based language agents. Reflexion proposes that language agents can improve through verbal feedback and episodic memory without updating model parameters [@shinn2023reflexion]. Voyager introduces an embodied lifelong learning agent that stores executable skills in a growing skill library and reuses them to solve new tasks [@wang2023voyager]. Generative Agents use memory streams, reflection, and planning to produce believable long-horizon agent behavior [@park2023generative].

These works show that memory and reflection can improve the behavior of LLM-based agents across trials. EG-RSA adopts a similar high-level intuition but applies it to a different object: reward search. Instead of storing natural language reflections or action skills, EG-RSA stores reward-edit outcome lessons. Each lesson may include the previous schema, the edited schema, the edit plan, metric deltas, failure modes, hack risk changes, and rollback decisions. This makes the memory directly actionable for future reward editing.

Therefore, EG-RSA can be viewed as a memory-augmented reward designer. The LLM is not simply asked to generate a new reward from scratch in each iteration. Instead, it receives structured feedback about the current policy, retrieves relevant past reward-edit outcomes, and proposes constrained edits under the influence of accumulated experience.

## 2.4 Reward Hacking and Risk-Aware Reward Optimization

Reward hacking and reward misspecification are major concerns in reinforcement learning. When the proxy reward differs from the intended objective, an agent may exploit the reward function while failing the true task [@amodei2016concrete]. Prior work on reward misspecification shows that more capable agents can sometimes obtain higher proxy reward while achieving lower true reward, exposing a gap between optimized reward and intended behavior [@pan2022effects]. This issue is especially relevant for LLM-generated rewards, because generated reward functions may contain dense shaping terms that are easy to optimize but only loosely aligned with the final task objective.

EG-RSA addresses this risk by introducing failure-mode diagnosis, hack detection, behavior risk audit, and rollback. For example, if a policy receives high shaped reward but does not trigger terminal success, the framework can mark this as a shaping-goal mismatch. If contact-related signals are repeatedly exploited, it can identify repeated event exploitation or unstable contact behavior. These diagnostics are then combined with reward attribution to determine whether the current reward schema is dominated by dense guidance, stability quality, terminal success, safety constraints, or control costs.

At the same time, our experiments show that risk-aware auditing must be designed carefully. If the audit mechanism is too conservative, it may block the very edits needed to escape a low-success regime. In our LunarLander experiments, a strict medium-risk blocking rule under weak success evidence caused an audit-induced search deadlock: the system needed to edit dense guidance and terminal rewards to discover successful landing behavior, but such edits were repeatedly blocked as medium-risk changes. This motivates a more flexible risk-budget mechanism in which high-risk edits remain blocked, while medium-risk edits under weak success can be recorded as warnings and validated through training, post-hoc evaluation, and rollback.

## 2.5 Positioning of EG-RSA

In summary, prior work has made significant progress in reward shaping, inverse reinforcement learning, LLM-based reward generation, and memory-based language agents. EG-RSA combines these ideas but targets a distinct problem: how to make LLM-assisted reward design history-aware, interpretable, and auditable.

The key difference is that EG-RSA treats reward design as an iterative search process over structured reward schemas. It introduces semantic role attribution to explain which reward components drive policy behavior, stores effective and regressive reward edits as outcome lessons, constrains LLM modifications to auditable operators, and uses risk-aware audit and rollback to reduce reward hacking. This shifts the role of the LLM from a one-shot reward generator to an experience-guided reward search operator.
